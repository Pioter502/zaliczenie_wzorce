Celem tego wzorca projektowego jest udostępnienie jednolitego interfejsu umożliwiającego iterowanie po elementach znajdujących się wewnątrz danej klasy.
Zastosowanie takiego wzorca to głównie ukrywanie skomplikowanej struktury klasy przed użytkownikiem dla wygody lub bezpieczeństwa.
Stosuje się go również w celu redukcji duplikowania kodu przeglądania elementów zbiorów na przestrzeni całego programu lub
gdy piszemy kod ale nie jesteśmy pewni szczegółów tych struktur, ponieważ mogą być jeszcze nieokreślone.
